# 电子信息系耗材管理系统 — 原型截图与操作说明

> 原型文件：`prototype/index.html`
> 访问方式：`http://localhost:8000`（服务器启动后）
> 演示账号：admin / teacher01 / approver01，密码均为 123456

---

## 一、系统架构

```
浏览器 ──→ http://localhost:8000 ──→ FastAPI Server ──→ SQLite
   │                                        │
   └── 前端原型（SPA单页应用）               └── 35+ REST API
       纯HTML/CSS/JS + Chart.js                 Swagger文档：/docs
```

---

## 二、页面清单与操作流程

### 2.1 登录页

**界面元素**：用户名输入框、密码输入框、登录按钮  
**操作**：输入账号密码 → 点击「登录」→ 调用 `/api/auth/login` → 获取JWT Token → 进入工作台  
**角色差异**：不同角色登录后侧边栏菜单不同（教师看不到审批管理）

---

### 2.2 工作台（Dashboard）

**数据来源**：`/api/reports/stock` + `/api/materials` + `/api/requisitions`

| 区域 | 内容 | 数据来源 |
|------|------|----------|
| 统计卡片 | 耗材种类、库存总价值、预警项数、待审批数 | API 实时加载 |
| 消耗趋势图 | 柱状图（Chart.js） | 静态样例数据 |
| 类别占比图 | 环形图（Chart.js） | 静态样例数据 |
| 最近领用记录 | 5条最新申请 | `/api/requisitions` |
| 库存预警 | 低库存耗材列表 | `/api/materials?alert_status=low` |

---

### 2.3 入库登记

**操作流程**：
1. 填写入库信息（名称、规格、类别、数量、单价、供应商、批次号、位置）
2. 系统自动校验必填字段
3. 点击「确认入库」
4. 后端自动处理：
   - 检查耗材是否已存在（同名同规格 → 复用）
   - 不存在则自动创建新耗材（`POST /api/materials`）
   - 创建入库记录（`POST /api/inbound-records`）
   - 库存自动累加
5. 弹出成功提示，重置表单

---

### 2.4 库存查询

**操作流程**：
1. 页面加载时自动从 `/api/materials` 获取数据
2. 支持按关键词搜索、按类别筛选、按预警状态筛选
3. 低库存红色标注、高库存黄色标注
4. 点击「调整」→ 弹窗输入调整数量和原因

---

### 2.5 领用申请

**操作流程**：
1. 填写用途说明
2. 选择需要领用的耗材（支持添加多行）
3. 填写申请数量
4. 点击「提交申请」
5. 后端校验库存 → 创建申请记录 → 状态设为 pending
6. 等待审批人审核

---

### 2.6 审批管理

**操作流程**：
1. 审批人登录后可见
2. 待审批列表自动从 `/api/requisitions?status=pending` 加载
3. 点击「同意」→ 状态变为 approved，通知管理员出库
4. 点击「驳回」→ 状态变为 rejected，通知申请人
5. 点击「退回」→ 状态变为 returned，申请人可修改后重新提交

---

### 2.7 供应商管理

**操作流程**：
1. 列表从 `/api/suppliers` 加载
2. 显示供货次数、累计金额
3. 搜索框支持按名称筛选

---

### 2.8 操作日志

**数据来源**：`/api/audit-logs`  
**特性**：所有入库、出库、审批、登录操作自动记录，不可删除

---

### 2.9 报表导出

6个报表卡片：入库记录、领用记录、库存报表、盘点差异、供应商统计、数据备份。当前后端已实现 `GET /api/reports/*` 系列接口。

---

### 2.10 系统设置

4个Tab：个人资料、审批规则（阈值配置）、库存预警设置、用户管理

## 三、响应式布局

| 分辨率 | 适配方式 |
|--------|----------|
| ≥1024px | 完整侧边栏 + 内容区 |
| 768-1024px | 侧边栏可折叠 |
| <768px | 侧边栏隐藏，顶部汉堡菜单打开，卡片/表单单列布局 |

| Toast | 操作反馈提示 | `showToast(msg, type)` |
