# 电子信息系耗材管理系统 — 数据库设计文档

> 对应脚本：`database/schema.sql` · `database/seed.sql`
> ER说明：`database/ER说明.md`
> 数据库：SQLite（开发）/ MySQL 8.0+（生产）
> 字符集：utf8mb4

---

## 1. 数据表清单

| 表名 | 说明 | 主键 | 记录数（种子） |
|------|------|------|:--:|
| users | 系统用户 | id (INT AUTO_INCREMENT) | 7 |
| categories | 耗材类别（树形） | id | 22 |
| suppliers | 供应商信息 | id | 5 |
| materials | 耗材信息主表 | id | 22 |
| inbound_records | 入库记录 | id | 6 |
| requisitions | 领用申请表 | id | 6 |
| requisition_items | 领用明细子表 | id | 10 |
| audit_logs | 操作日志 | id | 9 |

---

## 2. 核心字段说明

### 2.1 materials（耗材信息主表）

| 字段 | 类型 | 说明 |
|------|------|------|
| material_code | VARCHAR(20) UNIQUE | 编号：HC + YYYYMMDD + 4位流水 |
| name + spec | UNIQUE(name, spec) | 同名同规格只能建一条 |
| stock_qty | INT | 库存数量，由出入库操作自动维护 |
| safety_stock_min | INT | 安全库存下限，低于此值触发预警 |
| safety_stock_max | INT | 安全库存上限，高于此值触发预警 |

### 2.2 requisitions（领用申请表）

| 字段 | 类型 | 说明 |
|------|------|------|
| status | ENUM(8种状态) | 状态流转见下方状态机 |
| total_amount | DECIMAL(14,2) | 申请总金额，提交时自动计算 |
| approver_id | FK → users | 审批人，审批时填入 |
| delivered_at | DATETIME | 出库时间 |

---

## 3. 领用申请状态机

```
draft(草稿) → pending(待审批) ──→ rejected(已驳回) [终态]
                    │         ──→ returned(退回修改) → pending
                    │
                    └──→ approved(已审批)
                              │
                              └──→ delivered(已出库)
                                        │
                                        └──→ signed(已签收)
                                                  │
                                                  └──→ returned_stock(已归还) [终态]
```

## 4. 编号生成规则

| 实体 | 前缀 | 格式 | 示例 |
|------|------|------|------|
| 耗材编号 | HC | HC + YYYYMMDD + 4位流水 | HC202606150001 |
| 入库单号 | RK | RK + YYYYMMDD + 4位流水 | RK202606150001 |
| 申请编号 | LY | LY + YYYYMMDD + 4位流水 | LY202606150001 |
| 明细编号 | ITM | ITM + YYYYMMDD + 3位流水 | ITM20260615001 |
| 供应商编号 | SUP | SUP + YYYY + 3位流水 | SUP2024001 |

---

## 5. 关键约束

| 约束 | 说明 |
|------|------|
| 入库数量 > 0 | CHECK(quantity > 0) |
| 领用数量 > 0 | CHECK(req_quantity > 0) |
| 同名同规格唯一 | UNIQUE(name, spec) |
| 入库记录软删除 | deleted_at 字段标记，不物理删除 |
| 日志不可删除 | audit_logs 表无删除字段，只追加 |
| 出库前校验库存 | 存储过程 sp_confirm_delivery 中校验 |

---

## 6. 外键关系

```
materials.category_id  → categories.id
materials.supplier_id  → suppliers.id
inbound_records.material_id → materials.id
inbound_records.supplier_id → suppliers.id
inbound_records.operator_id → users.id
requisitions.applicant_id → users.id
requisitions.approver_id  → users.id
requisitions.deliverer_id → users.id
requisition_items.requisition_id → requisitions.id (CASCADE)
requisition_items.material_id    → materials.id
audit_logs.user_id → users.id (SET NULL)
```

---

## 7. 索引设计

高频查询字段均已建索引：
- `materials`: category_id, name, stock_qty, status
- `requisitions`: applicant_id, approver_id, status, created_at
- `inbound_records`: material_id, supplier_id, inbound_date
- `audit_logs`: user_id, action_type, created_at

## 8. 从SQLite迁移到MySQL

```sql
-- 主要差异：
-- 1. INTEGER AUTO_INCREMENT → INT AUTO_INCREMENT（已完成）
-- 2. SQLite的PRAGMA foreign_keys=ON → MySQL默认启用
-- 3. 触发器语法：DELIMITER // ... // DELIMITER ; 为MySQL语法，SQLite跳过
-- 4. JSON_TABLE → MySQL 8.0+支持，SQLite需用json_each()
```
